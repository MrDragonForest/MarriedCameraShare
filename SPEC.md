# 家庭图片分享系统 - 规格说明书

## 1. 项目概述

- **项目名称**: MarriedCameraShare (家庭影像)
- **类型**: 单页Web应用 (SPA)
- **核心功能**: 家庭成员上传、分享、查看照片的轻量级系统
- **目标用户**: 家庭成员

## 2. 技术方案

- **前端**: 原生 HTML5 + CSS3 + JavaScript (无框架，简化部署)
- **数据存储**: JSON文件 (users.json 存用户, posts.json 存图片记录)
- **图片存储**: 按日期目录存储 (images/YYYY-MM-DD/)
- **部署**: GitHub Pages

## 4. 功能规格

### 4.1 登录/注册

| 功能 | 描述 |
|------|------|
| 登录 | 用户名 + 密码 |
| 注册 | 用户名 + 密码 + 验证码 |
| 验证码 | 4位数字，5分钟内有效，防止频繁注册 |
| 频繁注册拦截 | 同一IP 5分钟内最多注册3次 |
| 记住登录 | localStorage 存储登录态 |

**用户数据结构 (users.json)**:
```json
{
  "users": [
    {
      "username": "zhangsan",
      "password": "md5 hash",
      "nickname": "张三",
      "createdAt": "2026-03-30T10:00:00Z"
    }
  ],
  "registerLogs": [
    {"ip": "127.0.0.1", "time": "2026-03-30T10:00:00Z"}
  ]
}
```

### 4.2 图片动态流

| 功能 | 描述 |
|------|------|
| 分组显示 | 按"天"分组，每组包含该日所有图片 |
| 显示内容 | 图片缩略图 + 上传用户 + 日期 + 标签 |
| 分页 | 每页20条（按天计算），日期倒序 |
| 筛选 | 上传人(下拉) + 日期(日期选择器) + 标签(输入) |
| 筛选逻辑 | 多条件AND查询 |

**图片记录数据结构 (posts.json)**:
```json
{
  "posts": [
    {
      "id": "uuid",
      "username": "zhangsan",
      "nickname": "张三",
      "date": "2026-03-30",
      "filename": "2026-03-30/abc123.jpg",
      "tags": ["旅行", "家庭"],
      "uploadTime": "2026-03-30T10:30:00Z"
    }
  ]
}
```

### 4.3 上传功能

| 功能 | 描述 |
|------|------|
| 上传入口 | 顶部显眼按钮 + 键盘快捷键 |
| 文件选择 | 支持点击 / 拖拽 / 多选 |
| 文件限制 | 仅图片 (jpg/png/gif/webp)，单文件≤10MB |
| 标签输入 | 文字输入，回车或点击添加标签 |
| 预览 | 上传前显示选择图片的缩略图 |
| 存储 | images/YYYY-MM-DD/随机文件名.jpg |

### 4.4 部署

- 使用 GitHub Pages
- 静态文件直接部署
- 路径: images/ 和 data/ 目录
